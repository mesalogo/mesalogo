# 图谱可视化页面访问指南

## 🎯 页面位置

图谱可视化页面已经添加到系统中，可以通过以下方式访问：

### 1. 通过导航菜单
1. 登录系统后，在左侧导航栏找到 **"角色管理"** 分组
2. 在该分组下可以看到：
   - 📊 **记忆管理** (`/memory`)
   - 🌐 **图谱可视化** (`/graph/visualization`) ← **新增**

### 2. 直接URL访问
- 直接在浏览器地址栏输入：`/graph/visualization`

## 🔧 功能特性

### 主要功能
- **图谱数据展示**: 基于vis-network的交互式图谱可视化
- **节点交互**: 点击节点查看详细信息
- **数据统计**: 实时显示节点和关系数量
- **分区筛选**: 支持按group_id筛选数据
- **数据导出**: 支持将图谱数据导出为JSON格式

## 📋 使用步骤

### 1. 确保图谱增强已启用
1. 进入 **设置 → 图谱增强设置**
2. 确保图谱增强功能已启用
3. 确保Graphiti服务正在运行

### 2. 访问图谱可视化
1. 点击左侧导航 **角色管理 → 图谱可视化**
2. 页面会自动加载配置信息

### 3. 加载图谱数据
1. (可选) 在分区ID输入框中输入特定的group_id
2. 点击 **"加载图谱"** 按钮
3. 等待数据加载完成

### 4. 交互操作
- **查看节点**: 点击图谱中的节点查看详细信息
- **拖拽节点**: 可以拖拽节点调整布局
- **缩放**: 使用鼠标滚轮缩放图谱
- **导出数据**: 点击"导出数据"按钮下载JSON格式的图谱数据

## 🔌 API接口

图谱可视化页面使用以下API接口：

- `GET /api/graph-enhancement/visualization/data` - 获取图谱数据
- `GET /api/graph-enhancement/visualization/info` - 获取数据库信息  
- `GET /api/graph-enhancement/visualization/config` - 获取配置信息

## 🚨 常见问题

### 1. 页面显示"图谱增强未启用"
**解决方案**: 
- 进入设置页面启用图谱增强功能
- 确保配置了正确的Neo4j连接信息

### 2. 加载图谱数据失败
**可能原因**:
- Neo4j数据库未运行
- Graphiti服务未启动
- 网络连接问题

**解决方案**:
- 检查Neo4j数据库状态
- 在图谱增强设置中启动Graphiti服务
- 查看浏览器控制台错误信息

### 3. 图谱显示为空
**可能原因**:
- 数据库中没有图谱数据
- 指定的group_id不存在

**解决方案**:
- 先通过记忆管理或其他方式添加一些数据
- 尝试不指定group_id加载所有数据

### 4. 依赖包错误
如果遇到 `vis-network` 或 `vis-data` 模块未找到的错误：

```bash
cd frontend
npm install vis-network vis-data
```
