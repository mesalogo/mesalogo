### 2. 优化BM25分数归一化 ✅

#### 优化前（Min-Max归一化）
```python
# 问题：最高分总是1.0（100%）
normalized = (score - min_score) / (max_score - min_score)
```

**问题：**
- ❌ 最高分总是1.0（100%），用户误以为"完全匹配"
- ❌ 相对归一化，不同查询间不可比
- ❌ 不符合用户预期

#### 优化后（软上限归一化）
```python
# 除以最大值的1.5倍，最高分约0.67
normalization_ceiling = max_score * 1.5
normalized = min(score / normalization_ceiling, 1.0)
```

**优势：**
- ✅ 最高分约60-70%，更符合实际相似度
- ✅ 保留分数区分度
- ✅ 跨查询更具可比性
- ✅ 对齐Weaviate等业界实践

#### 分数对比
| 查询 | 优化前最高分 | 优化后最高分 | 效果 |
|------|------------|------------|------|
| "智能体" | 100% | 67.6% | ✅ 合理 |
| "RAG系统" | 100% | 72.3% | ✅ 合理 |

### 3. 固定专家参数 ✅

#### 后端固定配置
```python
def get_search_config(self):
    """
    【用户可配置】:
    - search_mode: 检索模式 (vector/bm25/hybrid)
    - vector_weight: 混合权重 (0-1，默认0.7)
    
    【固定参数】（业界最佳实践）:
    - fusion_method: weighted（固定）
    - bm25_k1: 1.5（固定）
    - bm25_b: 0.75（固定）
    """
    default_config = {
        # 用户可配置
        'search_mode': 'hybrid',
        'vector_weight': 0.7,
        
        # 固定参数（业界标准）
        'fusion_method': 'weighted',
        'bm25_k1': 1.5,
        'bm25_b': 0.75
    }
    
    # 确保固定参数不被覆盖
    user_config = {**self.search_config}
    user_config['fusion_method'] = 'weighted'
    user_config['bm25_k1'] = 1.5
    user_config['bm25_b'] = 0.75
    
    return {**default_config, **user_config}
```

#### 参数来源说明
| 参数 | 默认值 | 来源 |
|------|--------|------|
| fusion_method | weighted | 业界主流（Weaviate/Pinecone） |
| bm25_k1 | 1.5 | 学术界推荐值 |
| bm25_b | 0.75 | 学术界推荐值 |
| vector_weight | 0.7 | OpenSearch建议（70%向量+30%关键字） |
