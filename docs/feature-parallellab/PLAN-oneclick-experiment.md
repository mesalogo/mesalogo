# 一键创建并行实验 - 实现计划

## 背景

参考现有的"一键创建行动任务"功能，设计"一键创建并行实验"功能，降低用户使用门槛。

## 现有一键创建行动任务的流程

```
需求输入 → AI生成角色 → AI生成行动空间 → AI生成规则 → AI生成任务 → 创建所有内容
```

**特点**：5个步骤，每步都需要 AI 生成，用户可编辑后确认。

## 一键创建并行实验的设计

### 核心思路：KISS 原则

并行实验**依赖已有的行动空间**，不需要从零创建。因此可以大幅简化：

```
选择行动空间 → 输入实验目标 → AI生成实验配置 → 确认并创建
```

**只需 3 个步骤**，比一键创建行动任务更简单。

---

## 详细设计

### 步骤 1：选择行动空间 + 输入实验目标

用户需要：
1. 从下拉框选择一个已有的行动空间
2. 输入实验目标/研究问题（自然语言描述）

**示例输入**：
```
我想研究不同的服务态度（热情/中立/冷淡）对客户满意度的影响
```

### 步骤 2：AI 生成实验配置 + 优化建议

AI 根据用户输入自动生成：
1. **实验名称**：基于目标自动命名
2. **讨论主题**：基于目标生成
3. **扫描变量**：从行动空间变量中选择，并生成扫描值
4. **目标变量**：识别要优化的指标
5. **实验协议**：生成行为协议
6. **优化建议**：分析行动空间，提供改进建议

**AI 生成的配置示例**：
```json
{
  "name": "服务态度对满意度影响实验",
  "topic": "模拟客服与客户的对话，研究不同服务态度对客户满意度的影响",
  "variables": {
    "service_attitude": {
      "type": "enumerated",
      "values": ["热情", "中立", "冷淡"]
    }
  },
  "objectives": [
    {
      "variable": "customer_satisfaction",
      "type": "maximize",
      "description": "客户满意度评分"
    }
  ],
  "experiment_protocol": "...",
  "suggestions": [
    {
      "type": "add_role",
      "title": "建议添加角色",
      "description": "当前行动空间只有客服角色，建议添加「客户」角色以模拟真实对话场景",
      "action": "添加客户角色"
    },
    {
      "type": "add_variable",
      "title": "建议添加变量",
      "description": "可以添加「响应时间」变量，研究响应速度对满意度的影响",
      "action": "添加 response_time 变量"
    },
    {
      "type": "adjust_config",
      "title": "建议调整配置",
      "description": "当前并发数为3，建议增加到5以获得更多样本",
      "action": "调整并发数"
    }
  ]
}
```

**优化建议类型**：
| 类型 | 说明 | 示例 |
|------|------|------|
| `add_role` | 建议添加角色 | "市场价格模拟中可能需要增加消费者角色" |
| `add_variable` | 建议添加变量 | "可以添加价格敏感度变量" |
| `modify_variable` | 建议修改变量 | "建议将价格范围从 [10,20] 扩大到 [5,30]" |
| `add_objective` | 建议添加目标 | "除了利润，还可以关注市场份额" |
| `adjust_config` | 建议调整配置 | "建议增加实验轮数以获得更稳定结果" |
| `warning` | 潜在问题警告 | "当前变量组合数过多(100+)，可能需要较长时间" |

### 步骤 3：确认并创建

用户可以：
1. 预览生成的配置
2. 查看并采纳优化建议（可选）
3. 编辑任意字段
4. 确认创建实验
5. 可选：直接启动实验

**优化建议的交互方式**：
- 建议以卡片形式展示在配置下方
- 每个建议有"采纳"和"忽略"按钮
- 采纳后自动应用到配置中（如添加角色会跳转到行动空间编辑）
- 忽略后建议卡片折叠

## 技术实现

### 前端组件

```
frontend/src/components/OneClickExperiment/
├── OneClickExperimentModal.tsx   # 主弹窗组件
└── index.ts                      # 导出
```

### 后端 API

```
POST /api/parallel-experiments/generate-config
请求：
{
  "action_space_id": "xxx",
  "experiment_goal": "用户输入的实验目标"
}

响应：
{
  "success": true,
  "config": {
    "name": "...",
    "topic": "...",
    "variables": {...},
    "objectives": [...],
    "experiment_protocol": "...",
    "suggestions": [...]
  }
}
```

### 提示词模板

新增提示词模板 `experimentConfigGeneration`：

```
你是一个实验设计专家。根据用户的实验目标和行动空间信息，生成并行实验配置，并提供优化建议。

## 行动空间信息
名称：{action_space_name}
描述：{action_space_description}
角色列表：{roles_list}
可用变量：{variables_list}

## 用户实验目标
{experiment_goal}

## 输出要求
请生成以下 JSON 格式的实验配置：
{
  "name": "实验名称（简洁明了）",
  "topic": "讨论主题（详细描述实验场景）",
  "variables": {
    "变量名": {
      "type": "enumerated|stepped|random",
      "values": [...] // 枚举值
      // 或 "start": x, "step": y, "end": z // 步进值
    }
  },
  "objectives": [
    {
      "variable": "目标变量名",
      "type": "maximize|minimize",
      "description": "目标说明"
    }
  ],
  "rounds": 3,
  "concurrency": 3,
  "experiment_protocol": "实验行为协议（Markdown格式）",
  "suggestions": [
    {
      "type": "add_role|add_variable|modify_variable|add_objective|adjust_config|warning",
      "title": "建议标题",
      "description": "详细说明为什么需要这个改进",
      "action": "具体操作建议"
    }
  ]
}

## 优化建议指南
分析行动空间时，请考虑以下方面：
1. 角色完整性：实验场景是否需要更多角色？（如市场模拟需要买家和卖家）
2. 变量覆盖：是否有重要的影响因素未被纳入变量？
3. 目标全面性：除了主要目标，是否有其他值得关注的指标？
4. 配置合理性：参数组合数是否过多/过少？轮数是否足够？
5. 潜在问题：是否有可能导致实验失败的配置问题？
```

## 入口位置

### 方案 A：并行实验室列表页（推荐）

在"创建实验"按钮旁边添加"一键创建"按钮：

```
[创建实验]  [一键创建]
```

### 方案 B：ActionTask 详情页

在 ActionTask 详情页添加"创建并行实验"按钮，点击后打开一键创建弹窗，自动填充行动空间。

### 方案 C：两者都有

同时在两个位置提供入口。

## 与现有功能的对比

| 功能 | 一键创建行动任务 | 一键创建并行实验 |
|------|------------------|------------------|
| 步骤数 | 5 步 | 3 步 |
| AI 调用次数 | 4 次 | 1 次 |
| 依赖 | 无 | 需要已有行动空间 |
| 复杂度 | 高 | 低 |
| 用户输入 | 需求描述 | 行动空间 + 实验目标 |
