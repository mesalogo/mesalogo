# 菜单重组设计方案

参考阿里云和AWS控制台的设计理念，对现有菜单进行重组优化。

## 一、设计原则

### 1. 扁平化结构
- **最多2层菜单**：减少导航复杂度
- **合并相关功能**：将分散的功能集中到统一视图
- **按使用频率组织**：高频功能前置

### 2. 视觉分组
- **使用分组标题**：不可点击的分类标签
- **视觉分隔**：用分隔线或留白区分不同业务域
- **图标语义化**：一眼识别功能类型

### 3. 渐进式展示
- **角色权限控制**：根据用户角色显示菜单
- **收藏/最近访问**：个性化快捷入口
- **搜索功能**：快速定位功能

## 二、重组后的菜单结构

### 🎯 方案A：业务导向型（推荐）

```
顶部导航栏
├─ Logo + 产品名
├─ 全局搜索框 [NEW]
├─ 任务中心/作业队列 
├─ 消息通知 [NEW]
├─ 语言切换
└─ 用户下拉菜单

左侧菜单
├─ 📊 工作台                    [一级菜单，Dashboard概览]
│
├─ ━━━━━━ 核心功能 ━━━━━━       [分组标题，灰色不可点击]
├─ 🎯 任务中心                   [合并：任务管理 + 并行实验室]
├─ 👥 智能体                     [合并：角色管理 + 能力工具 + 知识库]
├─ 🌐 行动空间                   [合并：行动空间 + 联合空间 + 规则]
├─ 📈 运行监控                   [监控 + 工作空间浏览器]
│
├─ ━━━━━━ 资源管理 ━━━━━━       [分组标题]
├─ 💾 知识库                     [独立出来，高频功能]
├─ 🧠 记忆管理                   [Memory]
├─ 🔧 工具市场                   [能力工具市场化]
│
├─ ━━━━━━ 配置与运维 ━━━━━━     [分组标题]
├─ ⚙️ 系统配置                   [仅管理员：基础配置+模型配置]
├─ 👤 用户管理                   [仅管理员]
├─ 🔌 集成服务                   [MCP服务器 + 图增强]
├─ 📋 运行日志                   [仅管理员]
│
└─ ℹ️ 关于                      [底部固定]
```

**核心改进：**
- ✅ 从20+菜单项 → 12个主菜单项
- ✅ 取消二级展开，点击进入后用Tab切换
- ✅ 分组用分隔线+标题，不占用菜单项
- ✅ 核心功能前置，配置类后置

## 四、具体页面内Tab优化

点击一级菜单后，在页面内部使用Tab切换细分功能：

### 示例：点击"智能体"菜单

```
页面布局：

┌─────────────────────────────────────────┐
│  👥 智能体管理                            │  [页面标题]
├─────────────────────────────────────────┤
│  [角色] [能力工具] [知识库] [记忆]         │  [Tab导航条]
├─────────────────────────────────────────┤
│                                         │
│          当前Tab的内容区域                │
│                                         │
└─────────────────────────────────────────┘
```

**Tab样式：**
```javascript
Tabs {
  type: 'card',             // 卡片式Tab
  size: 'large',
  style: {
    marginBottom: '24px',
    background: '#ffffff',
    padding: '16px 24px 0',
    borderRadius: '8px 8px 0 0'
  }
}
```

## 五、交互优化

### 1. 全局搜索
在顶部添加搜索框，输入关键词快速跳转：
- 支持功能名称搜索
- 支持拼音首字母搜索
- 显示最近访问记录
- 显示搜索结果分类

### 2. 面包屑导航
在页面顶部显示当前位置：
```
首页 > 智能体管理 > 角色管理
```

### 3. 快捷操作面板
在工作台页面提供：
- 常用功能卡片
- 最近访问列表
- 快速创建按钮
- 数据统计概览

### 4. 响应式收起
- 侧边栏收起后显示图标
- Hover时显示Tooltip提示
- 记住用户的收起状态

- 20+ 菜单项，3层结构

### 改进后

- 收藏+最近访问
- 全局搜索快速定位

## 八、推荐方案

**推荐：方案A（业务导向型）**

**理由：**
1. ✅ 平衡了功能组织和用户习惯
2. ✅ 实施难度适中，不需要大改
3. ✅ 扁平化明显，导航效率提升
4. ✅ 视觉清晰，符合现代设计趋势
5. ✅ 保留左侧菜单，符合国内用户习惯

**关键改进点：**
- 菜单项从20+减少到12个
- 使用分组标题代替可点击的分组
- 页面内用Tab切换细分功能
- 添加全局搜索和快捷入口
