# 监督者干预功能测试指南

## 测试目标
验证监督者干预功能是否能够正确地在任务会话中显示用户消息和智能体的实时流式输出。

## 测试步骤

### 1. 准备工作
1. 确保后端服务运行在 `http://localhost:8080`
2. 确保前端服务运行在 `http://localhost:3000`
3. 确保数据库中有包含监督者智能体的任务

### 2. 创建测试任务（如果需要）
```bash
# 创建一个包含监督者智能体的任务
curl -X POST "http://localhost:8080/api/action-tasks" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "监督者干预测试任务",
    "description": "用于测试监督者干预功能",
    "action_space_id": 1,
    "agents": [
      {"id": 1, "name": "普通智能体", "is_observer": false},
      {"id": 13, "name": "监督者智能体", "is_observer": true}
    ]
  }'
```

### 3. 前端测试步骤

#### 步骤1：打开任务详情页
1. 访问 `http://localhost:3000/action-tasks/1`（替换为实际任务ID）
2. 确保页面正常加载，显示任务信息

#### 步骤2：检查监督会话tab
1. 点击右侧边栏的"监督会话"tab
2. 确保能看到监督者智能体选择器
3. 确保能看到发送目标选择器（监督会话/任务会话）

#### 步骤3：测试监督者干预
1. 在监督会话tab中：
   - 选择一个监督者智能体
   - 选择发送目标为"任务会话"
   - 输入测试消息："这是一个监督者干预测试消息"
   - 点击发送

#### 步骤4：验证结果
1. **用户消息应该立即出现在左侧任务会话中**
2. **监督者智能体应该开始实时流式输出**
3. **流式输出应该显示在左侧任务会话中**
4. **同时在右侧监督会话tab中也应该有记录**

### 4. 预期行为

#### 正确的行为：
- ✅ 用户消息立即出现在任务会话
- ✅ 监督者智能体实时流式输出
- ✅ 完全等同于在任务会话中手动选择监督者发送消息
- ✅ 监督会话tab中保留完整记录

#### 错误的行为：
- ❌ 消息只出现在监督会话tab中
- ❌ 需要刷新页面才能看到消息
- ❌ 没有实时流式输出
- ❌ 出现错误提示

### 5. 调试信息
如果功能不正常，检查浏览器控制台的以下日志：
- "处理监督者干预，委托给任务会话组件"
- "监督者干预：模拟任务会话发送消息"
- "监督者干预消息发送完成"

### 6. 技术实现验证
监督者干预的技术流程：
1. 监督者组件检测到 `send_target === 'task_intervention'`
2. 调用父组件的 `handleSupervisorIntervention` 回调
3. 父组件通过 ref 调用任务会话组件的 `sendSupervisorIntervention` 方法
4. 任务会话组件临时设置目标智能体为监督者
5. 调用现有的 `sendMessage` 逻辑
6. 恢复原始状态

这样实现的效果完全等同于用户在任务会话中手动选择监督者智能体发送消息。
